# Operator Sweep

The operator sweep is a release-time check for VoxKitchen's Docker images. It
runs each built-in operator through a small YAML pipeline in the image that
users would normally use, then writes a Markdown report to
`scripts/sweep/last-run.md`.

This is intentionally separate from regular unit tests. Unit tests keep local
development fast; the sweep catches image-level drift such as missing runtime
dependencies, stale operator code inside a published image, or operators that
start successfully but produce unusable artifacts.

`last-run.md` is the latest local report generated by the maintainer. It is a
local evidence file and is gitignored. It may come from a filtered run, so read
the verdict line before treating it as a full-release signal.

Use `--report <path>` when you want to keep multiple filtered reports. Relative
paths are resolved from the repository root.

## Common Commands

Generate derived fixtures:

```bash
python scripts/sweep/run.py --setup
```

Run every sweep pipeline, pulling missing images when needed:

```bash
python scripts/sweep/run.py
```

Run one image family:

```bash
python scripts/sweep/run.py --image slim
python scripts/sweep/run.py --image asr
```

Run one image family and save its report separately:

```bash
python scripts/sweep/run.py --image slim --cleanup-each --report scripts/sweep/reports/slim.md
```

Run one operator:

```bash
python scripts/sweep/run.py --op wenet_asr
```

Use only images that are already local:

```bash
python scripts/sweep/run.py --no-pull
```

Re-render `last-run.md` from existing `work/vkit-sweep/<op>` directories
without launching Docker:

```bash
python scripts/sweep/run.py --report-only
```

Re-render a custom report path:

```bash
python scripts/sweep/run.py --report-only --report scripts/sweep/reports/local.md
```

## Release Checklist

Before cutting a release or publishing refreshed Docker images:

```bash
python scripts/sweep/run.py --setup
python scripts/sweep/run.py
```

For gated models such as Pyannote, pass a Hugging Face token:

```bash
HF_TOKEN=... python scripts/sweep/run.py
```

Attach the `scripts/sweep/last-run.md` verdict summary to the release notes.
If a run is intentionally filtered with `--op` or `--image`, say so next to the
summary.

When running image families sequentially to keep disk use low, write each report
to a separate path so later runs do not overwrite earlier results:

```bash
python scripts/sweep/run.py --image slim --cleanup-each --report scripts/sweep/reports/slim.md
python scripts/sweep/run.py --image asr --cleanup-each --report scripts/sweep/reports/asr.md
python scripts/sweep/run.py --image diarize --cleanup-each --report scripts/sweep/reports/diarize.md
python scripts/sweep/run.py --image tts --cleanup-each --report scripts/sweep/reports/tts.md
python scripts/sweep/run.py --image fish-speech --cleanup-each --report scripts/sweep/reports/fish-speech.md
```

## Adding Or Changing Operators

Every registered operator should have a matching
`scripts/sweep/pipelines/<op>.yaml` file. The coverage gate in
`tests/unit/sweep/test_sweep_coverage.py` enforces this so new operators do not
silently miss image-level coverage.

When an operator needs a semantic assertion beyond "the pipeline produced final
cuts", add it to `scripts/sweep/assertions.py`.
